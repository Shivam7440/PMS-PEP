# MERN Stack Project

This project contains a MERN (MongoDB, Express, React, Node.js) stack application completely Dockerized for local development and ready for production deployment on Render.

## Architecture

The application is split into two main services:
1.  **Backend (`/backend`)**: A Node.js Express server that interacts with a MongoDB database (configured via external URI) and provides a REST API.
2.  **Frontend (`/frontend`)**: A React application built with Vite. In the production Docker container, it is built into static assets and served using Nginx.

## Requirements

- [Docker](https://docs.docker.com/get-docker/)
- [Docker Compose](https://docs.docker.com/compose/install/)

## Steps to Run Locally

1. **Clone the repository** (if you haven't already).
2. **Environment Variables**: Ensure your `backend/.env` file exists and is populated with your MongoDB URI, JWT secrets, Cloudinary credentials, and email configuration.
3. **Build and Run the Containers**:
   Use Docker Compose to build the images and start the containers in detached mode:
   ```bash
   docker-compose up -d --build
   ```
4. **Access the Application**:
   - Frontend (React): `http://localhost:3000`
   - Backend (Node.js API): `http://localhost:5001`
5. **Stop the Containers**:
   ```bash
   docker-compose down
   ```

### Docker Commands Used
- `docker-compose up -d --build`: Builds the Docker images and starts the containers in detached mode.
- `docker-compose down`: Stops and removes the containers, networks, and volumes created by `up`.
- `docker-compose logs -f`: Tails the logs from all running services.

## Deployment

The backend of this application is configured to be deployed easily to [Render](https://render.com) using the included `render.yaml` Blueprint file.

### Deployment Link
*(Since the deployment is connected to your personal GitHub, the live link will be generated by Render upon deployment. Once deployed, the public URL provided by Render will act as your deployment link.)*

### Steps to Deploy Backend to Render
1. Push this repository (including the `render.yaml` and Dockerfiles) to your GitHub account.
2. Log in to [Render.com](https://dashboard.render.com).
3. Click on the "New +" button and select "Blueprint".
4. Connect your GitHub account and select this repository.
5. Render will automatically detect the `render.yaml` file.
6. Important: During the setup, Render will prompt you to provide values for the sync: false environment variables (like `MONGODB_URI`, `JWT_SECRET`, etc.). You must copy these values from your local `backend/.env` file.
7. Click "Apply" to start the deployment process. Render will build the Docker container using `backend/Dockerfile` and deploy your web service.
